# Q：为什么已经编写完节点图并且保存了，在想要在节点图配置界面里进行挂载的时候找不到我写好的节点图？

A：节点图有不同的类型页签，请确保您编写的节点图位置是否正确；如需要挂载在【实体】上的节点图，请在【实体节点图】页签下编写并保存，才能在实体上挂载对应节点图；如果您在【状态节点图】页签下编写保存的节点图，是无法在【实体】节点图配置界面中找到的。

# Q：为什么我的节点图循环/运算执行到一半就停止执行了？

A：“千星沙箱”中，当存在计算量较大的节点时（如循环节点，或者一些节点比较多的节点图），其执行次数可能存在上限，当奇匠需要编辑此类节点图时，可以将其拆分；后续更新中会增加功能：当计算量达到上限时，千星沙箱会出现提示；

# Q：我不小心把千星沙箱整个窗口关掉了，应该怎么再次打开？

A：在Esc界面左下角，点击千星沙箱图标后等待几秒后，千星沙箱便会打开。

# Q：我怎么获取不到角色的GUID？

A：角色本身没有GUID，因此请注意在节点图GUID参数获取中，不要获取角色的GUID，这会导致功能不运行。

# Q：【获取自身实体】获取的是什么？是玩家、角色还是实体？

A：节点图挂载在什么实体身上，【获取自身实体】获取的就是什么实体，如挂载在物件实体上，那么此实体上的节点图【获取自身实体】传出的参数就是此物件实体的实体数据；如果挂在角色上，那么此角色上的节点图【获取自身实体】传出的参数就是此角色的实体数据，玩家同理。

# Q：【玩家】和【角色】之间是什么关系？为什么有些实体引脚输入玩家/角色的实体参数不生效？

A：在局内编辑器中，【玩家】：玩家是服务器概念上的独立个体，即每个进入游戏的玩家，每名玩家的变量存储也都独立存在；【角色】：是由玩家进行操作控制的特殊实体，在关卡中与各类组件进行交互的实体都是【角色】；因为两者是不同的实体，因此在编辑节点图中，需要严格区分出入参的实体是【角色】还是【玩家】，两者混淆会导致逻辑无法正常跑通。

同时需要注意，玩家有固定的GUID，角色没有GUID。

# Q：为什么【传送玩家】节点有时会出loading，有时又直接瞬移？

A：传送玩家在传送距离与原位置超出一定数值时，会进入Loading状态。如果从传送位置较近，则直接瞬移到目标位置。

# Q：为什么【传送玩家】节点不被正常执行？

A：在保证被传送玩家正常入参的情况下， 可能是因为传送坐标超出了场景生效范围导致的，可以在Esc界面—【关卡设置】中将场景生效范围扩大。

# Q：为什么在【获取列表对应值】中，【序号】填“1”获取不到参数/获取到的是列表里第2个参数？

A：在局内编辑器中，列表采用的是零基索引。这就导致在给列表出入参时，第一个参数的序号是0。比如在获取在场的第一个玩家时，【获取列表对应值】的索引应该填写“0”。

但同时也要注意，并非所有地方都采用了零基索引，如在组件中，【选项卡】等索引就是以“1”为初始值，需要注意区分。

# Q：为什么有时候【自定义变量变化时】事件监听不到变量变化？

A：首先请确认目标变量是否正常变化，可以用【打印字符串】节点在变量变化之后输出该变量值来确认是否变化。

如果没有正常变化，可以

①检查【设置自定义变量】是否定义了【目标实体】；

②检查目标实体是否已经挂载自定义变量组件且有对应名称的变量槽位；

③检查变量数据类型是否匹配，否则变量变化是不生效的。

④给用执行节点给自定义列表插入值是不触发自定义变量变化事件的。

如果确认变量正常变化，但是依旧监听不到变量变化的事件，同样请确认监听自定义变量变化的变量数据类型是否匹配。保证从发出、查询、监听所有节点的数据类型都和目标自定义变量数据类型相等。如检查【变化后值】【变化前值】中的数据类型定义是否和要监听的变量数据类型相等，以及改变、获取变量时的数据类型都保持一致。

此外还需要检查【设置自定义变量】执行节点中的“是否触发事件”勾选了【是】，否则不发出事件。

# Q：【全局计时器】和【定时器】有什么区别？

A：【全局计时器】指的是已经在【计时管理器】中配置的计时器，通过组件【全局计时器】可以调取，但必须提前配置；挂载之后，物体/元件上的全局计时器相关节点才能取参、生效。并且计时器可以被UI界面中的计时器直接取用。

【定时器】是可以通过节点图动态创建，挂载，停止，循环的一种时间轴定时器。可以结合拼装列表触发定时器事件，达到打点计时器或延时触发的效果；根据自身需要可以在两者中灵活取用。(如下图配置，在启动定时器之后，每1秒都能触发一次定时器触发时事件）

# Q：【启动定时器】是创建新的定时器还是启动已有定时器？

A：【启动定时器】是创建一个新定时器并立即启动；如需要重启一个被暂停的定时器，可以使用【恢复定时器】节点。

# Q：在节点图编辑【特效】中，怎么预览特效？

A：点击特效资产后的放大镜图标，之后可以点击列表中的播放按钮进行播放。此时打开实体摆放页面，就可以看到特效会在屏幕中间以默认大小播放。

# Q：为什么用节点【播放特效】创建特效之后，试玩里特效不显示？

A：①首先确认选择的元件/造物具有“播放特效”这一组件。

②其次触发特效的节点图逻辑挂载在对应元件/造物中，并且逻辑正确、能正常走到执行节点。

③然后确认特效资源ID为可用ID（播放特效中的特效ID为非循环特效，不要复制粘贴循环特效ID），目标实体正常入参。

④最后注意参数：目标实体正常挂接，同时挂接点为有效挂接点，同时缩放倍率不为0。

# Q：为什么用节点图挂载特效，特效位置和挂接点之间不是精准贴合？

A：现阶段使用节点图挂载特效时，特效不会精准贴合地挂在具体挂接点上。

# Q：节点引脚的数据类型是固定的吗？可以用整数0和1替代布尔值的True和Flase吗？

A：节点图中引脚的数值均为固定类型，当类型不符时，只会获得一个对应类型的默认值（例如，无法使用整数的0和1，来替代布尔值中的否和是）；请奇匠注意检查引脚的数值类型、必要时可以使用“数据类型转换”节点；

# Q：为什么“攻击命中时”事件不正常回传攻击者信息？

A：在“千星沙箱”中，使用“攻击命中时”节点时，若受击实体被某一次攻击击杀，由于实体已经销毁，节点会无法获取到本次攻击受击者的信息；

# Q：为什么销毁节点后的逻辑不生效？【实体销毁时】事件为什么不传参？造物（怪物）身上为什么监听不到销毁事件？

A：【销毁时】事件需要挂载在【关卡实体】（LevelEntity）上。这是由于：除了【关卡实体】（LevelEntity）的节点图外，实体销毁/移除时事件都不会生效，因为实体监听不到其他实体销毁，如果节点图中有销毁实体，那么在执行销毁实体后，此实体的所有逻辑都会失效。

# Q：我想在游戏开始/关卡创建时就触发事件执行逻辑，但是没有找到关卡开始时类似事件？

A：建议在关卡实体（LevelEntity）上挂载节点图，用【实体创建时】事件来实现此效果

# Q：我想动态设置某个实体/特效的位置/大小，应该怎么操作？

A：现阶段不支持通过节点图动态改变场上某个实体/特效的位置、大小，可以通过销毁+创建的方式来达到效果。

# Q：应该怎么拆分向量？怎么将1个向量参数拆分为3个可以单独入参的参数？

A：可以通过运算节点【拆分三维向量】、【创建三维向量】的方式将向量拆分和合并。

# Q：【创建实体】节点为什么没有位置的入参？我怎么确认它被正常创建了？

A：【创建实体】节点实际上是针对已经在实体摆放界面摆放过且关闭了【初始创建】的实体使用的。因此它只起到开关的作用。如果想要动态生成某个元件的实体，则应该改用【创建元件】节点来实现效果。

# Q：复合节点两个相同数据类型的引脚怎么进行合并？

A：在需要被合并的引脚上右键，选择【合并引脚】。

此时相同数据类型的引脚序号会变空，如下图。

随后点击变空的引脚，将其勾选后，再在“选择引脚合并至XXX”勾选确认即可完成合并。

# Q：【创建元件】节点是在元件库内创建一个新的元件吗？还是在实体摆放界面创建元件？

A：实际上【创建元件】节点是根据元件库中已编辑保存的元件，在实体摆放界面上创建它的子实体。创建出的实体继承元件的所有逻辑，但是每个实体单独处理自己身上的逻辑。

（如在实体摆放界面放置了若干个\[可以被吃到的金币\]元件的子实体，角色在吃到其中一个金币时，只会使得被吃到的特定金币消失，而非使所有\[可以被吃到的金币\]元件的子实体金币都消失）

# Q：【创建元件】后为什么获取不到创建实体的GUID？

A：在“千星沙箱”中，“创建元件”的节点创建的实体，其GUID是无法获取到的；

# Q：为什么玩家、角色身上挂载【状态变化时】【状态移除时】事件不正常生效？

A：这两个节点需要在【状态节点图】中创建节点图进行配置，且挂载在对应状态上才会生效，作为实体节点图挂载在玩家、角色、造物（怪物）、物体身上都不会生效。

# Q：为什么我已经动态将自身仇恨值降为0，但是造物（怪物）依然在攻击我？

A：您依然在造物（怪物）的仇恨列表中，可以通过将自身移除出仇恨列表来实现效果。

# Q：为什么修改追踪小地图表示的玩家列表这个节点无法运行?

A：节点“修改追踪小地图标识的玩家列表”，不支持追踪类型是“范围”的小地图标识。
