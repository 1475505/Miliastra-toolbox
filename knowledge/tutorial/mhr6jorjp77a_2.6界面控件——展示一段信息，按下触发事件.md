# 界面控件——玩家设备上的UI信息与按钮

**界面控件：** 通过配置控件和控件信息，制作在玩家设备上显示的UI信息或按钮 _（如：弹窗提示、进度条、属性信息、计时器、可交互道具等均为界面控件）_

## 功能示例-1：信息类控件-常驻显示的动态文本框

### 1.概念简述

**信息类控件：** 以展示信息为主，可展示配置固定/动态内容（固定-文字引导、动态-进度变化），【文本框】、【弹窗】、【进度条】、【全局计时器】、【计分板】都属于信息类控件

_\*\*教程中将以【文本框】进行讲解_

### 2.教学内容

**功能效果：** 制作一个显示积分信息的UI，拾取金币后积分+1

**效果演示：**

_\*\*演示中的文本显示，需达到特定奇匠等级后解锁【自定义文本外显】功能，教程中以效果展示为主_

### 3.操作步骤

**1.【局内编辑器】添加自定义变量**

在【玩家】上添加一个自定义变量（命名可自由配置），类型为整数，初始值为0

**2.【局内编辑器】制作控件**

点击局内编辑器左上角（或直接按ESC键）>>>呼出左侧面板-ESC菜单栏，点击>>>进入界面控件编辑界面，在【界面控件组库】中添加界面控件模版>>>选择【文本框】

**3.【局内编辑器】配置控件参数**

配置文本框显示信息，保存设置

为达到预期的功能效果教程中配置如下：

|     |     |
| --- | --- |
| 要做的功能 | 对应的配置项 |
| 固定显示信息为 积分值说明 | 【文本内容】直接输入“积分：” |
| 动态显示信息为玩家自身的积分值变化 | 【文本内容】插入变量为玩家自身-积分值（步骤1添加的自定义变量） |
| 让信息看着显眼一些 | 调整【字号】、【位置】、【对齐】等参数 |

_\*\*当前局内编辑器界面布局与各机型相关，需要进行逐一配置（教程中以移动端和PC端为例）_

**4.【局内编辑器】添加控件到界面布局**

将制作完成的控件添加到界面布局中（在步骤3中调整好各机型配置效果，此处添加后会自动同步，无需额外配置）

\*\*界面控件组库中制作的类似元件库中制作的元件，实际游玩时并不显示，需要添加至界面布局中才会显示

**5.【局内编辑器】绑定界面控件到具体玩家**

在战斗预设-职业中，选择布局

_\*\*此处教程中无玩家展示信息差异化（如：玩家A展示信息A，玩家B展示信息B），直接使用默认配置；如有特殊需要，可对不同职业进行独立配置_

**6.【局内编辑器】制作可拾取的金币**

制作金币元件，作为拾取道具，添加【组件-碰撞触发器】，调整碰撞区域

_\*\*该功能在前置课程-碰撞与交互中已使用，此处不再列举配置项_

**7.【千星沙箱】节点图配置**

在实体节点图中，添加并配置金币元件的节点图（设置参考见下图），为达到预期的功能效果教程中配置如下：

|     |     |     |     |
| --- | --- | --- | --- |
|  |  | 要做的功能 | 对应的配置项 |
| 功能<br>拾取金币增加积分 | 触发场景 | 进入范围后拾取 | 【事件节点】：进入碰撞触发器时（从外部进入，接触到配置的区域时触发） |
| 条件 | / | / |
| 结果-1 | 拾取后增加1积分 | 【执行节点】：使用设置自定义变量设置【拾取后积分值=当前积分值+1】 |
| 结果-2 | 拾取金币 | 【执行节点】：使用【销毁实体】制作拾取效果 |

**8.【局内编辑器】挂载节点图**

将配置的节点图挂载在金币元件上，保存元件设置

**9.【局内编辑器】放置元件，试玩体验**

将制作好的金币元件放置在场景中，进入试玩即可体验该功能

_\*\*更多相关信息可自行查阅【综合指南-文本框界面控件】_

## 功能示例-2：交互类控件-点击后触发事件

### 1.概念简述

**交互类控件：** 可以进行点按交互，交互后发出一个事件，【交互按钮】属于交互类控件

_\*\*实际用法与【碰撞触发器】、【选项卡】类似，都是用于触发事件，区别仅在于交互按钮可控性更强，可随时触发事件，多用于制作主动效果（如：使用道具、主动技能、打开背包等）_

### 2.教学内容

**功能效果：** 制作一个主动增加积分的按钮，每次按下时积分+1

**效果演示：**

_\*\*演示中的文本显示，需达到特定奇匠等级后解锁【自定义文本外显】功能，教程中以效果展示为主_

### 3.操作步骤

功能示例中的积分值UI和变量设置与功能示例-1完全一致，此处仅说明，不额外赘述，以讲解【交互按钮】特殊配置为主

**1.【局内编辑器】添加自定义变量**

在【玩家】上添加一个自定义变量（命名可自由配置），类型为整数，初始值为0

**2.【局内编辑器】制作控件**

点击局内编辑器左上角（或直接按ESC键）>>>呼出左侧面板-ESC菜单栏，点击>>>进入界面控件编辑界面，在【界面控件组库】中添加界面控件模版>>>选择【文本框】和【交互按钮】

**3.【局内编辑器】配置控件参数**

**4.【局内编辑器】添加控件到界面布局**

将制作完成的控件添加到界面布局中

**5.【局内编辑器】绑定界面控件到具体玩家**

## 功能示例-3：信息类控件-占点进度条

**功能说明：** 制作一个占点进度条，玩家进入占点区域后，进度条增加，占满5秒可完成占点，提示游戏胜利；中途离开占点区域，则停止占点进度

**效果演示：**

_\*\*演示中的文本显示，需达到特定奇匠等级后解锁【自定义文本外显】功能，教程中以效果展示为主_

**实现步骤：**

1.在【玩家】上，添加3个自定义变量，分别对应占点进度的【最大值】、【最小值】、【当前值】

变量1，命名：【占点最大值】（命名可自由配置）；类型：整数；初始值：100

变量2，命名：【占点最小值】（命名可自由配置）；类型：整数；初始值：0

变量3，命名：【占点当前值】（命名可自由配置）；类型：整数；初始值：0

2.制作进度条控件，选择合适的【类型】、【大小】、【样式】、【颜色】等参数；配置当前值、最大值、最小值，对应【玩家自身】对应的变量值，保存设置

3.将制作完成的控件添加到界面布局中，并绑定给对应的职业（玩家）

4.创建一个元件，作为占点平台

5.在该元件上，添加【组件-碰撞触发器】，选择合适的类型并配置能覆盖平台的触发范围，生效目标设置为仅角色

6.配置该元件和玩家的节点图（设置参考见下图），为达到预期的功能效果教程中配置如下：

**元件节点图**

_\*\*由于占点是一个持续增加的进度状态，并且随时间变化而改变值，因此使用【定时器】以一定频率增加进度值（进入区域，启动定时器开始增加占点值；离开区域，终止计时器占点值不再增加）_

|     |     |     |     |
| --- | --- | --- | --- |
|  |  | 要做的功能 | 对应的配置项 |
| 功能-1<br>进入占点区后，可增加占点值 | 触发场景 | 进入圆台范围后触发 | 【事件节点】：进入碰撞触发器时（从外部进入，接触到配置的区域时触发） |
| 条件 | / | / |
| 结果 | 开始增加积分 | 【执行节点】：启动定时器，并挂载在玩家身上<br>由于循环触发，因此选择【循环=是】<br>教程中设置【拼接列表=0.5s】为定时器以0.5s触发一次，可根据实际需要灵活调整（如：0.1s触发一次、1s触发一次等） |
| 功能-2<br>离开占点区后，停止增加占点值 | 触发场景 | 离开圆台范围后触发 | 【事件节点】：离开碰撞触发器时（从内部出去，接触到配置的区域时触发） |
| 条件 | / | / |
| 结果 | 不再增加积分 | 【执行节点】：终止定时器，终止之前挂载的定时器即可 |

**玩家节点图**

| 结果 | 增加占点值 | 【执行节点】：使用增加自定义变量改变当前值<br>由于最大值=100，5s可达到满值，因此占点值0.5s涨幅10（新值=当前值+10） |
