# 卡牌选择器——可进行卡片选择交互的选项弹窗

## 功能示例-1：3选1选项弹窗

### 1.概念简述

**卡牌选择器：** 一个特殊的界面控件类型，通过配置卡牌选择器和卡牌面板，制作自定义的交互卡片选项，可通过节点图触发开关

### 2.教学内容

**功能效果：** 制作3选1选项弹窗，玩家点击交互按钮后打开选项弹窗，选择后获得对应效果

**效果演示：**

_\*\*演示中的文本显示，需达到Lv.3后解锁【自定义文本外显】功能，教程中以效果展示为主_

### 3.操作步骤

**1.【局内编辑器】添加卡牌选择器**

点击局内编辑器左上角（或直接按ESC键）>>>呼出左侧面板-ESC菜单栏，点击>>>进入界面控件编辑界面，在【界面控件组库】中添加预制模版>>>选择【卡牌选择器】

**2.【局内编辑器】配置控件参数**

配置【页面设置】与【选项配置】的信息，并保存设置

为达到预期的功能效果教程中配置如下：

|     |     |
| --- | --- |
| 要做的功能 | 对应的配置项 |
| 选项提示信息 | 【页面设置-标题文本】直接输入“请在以下3项中选择1项” |
| 必须选择其中之一，不能放弃选项 | 【页面设置-可放弃选择】关闭 |
| 配置选项卡牌与卡牌文本 | 【选项配置-卡牌库设置-详情编辑】<br>根据实际所需，添加卡牌，并修改其“类型”、“图标”、“标题”、“描述”等信息 |

**3.【局内编辑器】添加触发开关**

教程中为了方便，以点击交互按钮打开弹窗为例

在【界面布局】中添加预制模版>>>选择【交互按钮】

**4.【局内编辑器】添加自定义挂接点（\*\*此处配置仅便于表现，与实际功能作用无关）**

为了便于教程演示，添加一个角色背部挂接点，配置合适的位置偏移，用于直接配置装备法器后的效果

**5.【千星沙箱】节点图配置**

在实体节点图中，添加并配置玩家的节点图（设置参考见下图），为达到预期的功能效果教程中配置如下：

**\*\*该节点图挂载在【玩家】上**

|     |     |     |     |
| --- | --- | --- | --- |
|  |  | 要做的功能 | 对应的配置项 |
| 功能-1<br>玩家点击交互按钮后，打开3选1弹窗 | 触发场景 | 点击按钮后，触发事件 | 【事件节点】：界面控件组触发时 |
| 条件 | / | / |
| 结果 | 打开3选1弹窗 | 【执行节点】：使用【唤起卡牌选择器】制作打开弹窗的效果<br>\*\*决策结果/选项对应列表=卡牌ID<br>\*\*决策数量上限/下限影响在本次选择中可选择的选项数量 |
| 功能-2<br>选择选项后，获得对应选项的效果 | 触发场景 | 选择选项后 | 【事件节点】：卡牌选择器完成事件 |
| 条件-1 | 完成的是对应的卡牌选择器 | 使用双分支判断卡牌选择器索引<br>\*\*存在多个卡牌选择器时需要进行判断 |
| 条件-2 | 是玩家选择完成影响 | 使用双分支判断完成原因<br>\*\*避免由于超时等其他原因导致触发事件 |
| 条件-3 | 判断选择的卡牌选项是哪个 | 使用多分支判断选择结果<br>\*\*因为存在多选结果，返回的是列表，通过取列表的值判断 |
| 结果 | 根据选择的卡牌执行对应结果 | 教程中以添加武器特效为例，实际制作时可按照需求自由制作不同效果 |

**6.【局内编辑器】挂载节点图**

将配置的节点图挂载在玩家上

**7.【局内编辑器】试玩体验**

进入试玩即可体验该功能

_\*\*更多相关信息可自行查阅【综合指南-卡牌选择器】_

## 功能示例-2：4选2选项弹窗（含随机效果制作）

**功能说明：** 开启宝箱后，弹出4选2弹窗，玩家点击交互按钮后打开选项弹窗，选择后获得对应效果；其中有一个卡片为随机效果（通过节点图自定义其逻辑）

**效果演示：**

_\*\*随机卡片效果为：进行生命值扣除，扣除值为随机数_

_\*\*演示中的文本显示，需达到Lv.3后解锁【自定义文本外显】功能，教程中以效果展示为主_

**实现步骤：**

1.在【界面控件组库】中添加预制模版>>>选择【卡牌选择器】，添加新的卡牌选择器

2.添加【玩家】自定义变量

> 名称：获得的未知卡片效果；
>
> 类型：整数；
>
> 初始值：0

3.在【界面控件组库】中添加预制模版>>>选择【文本框】，作为选择隐藏卡片的效果提示，展示信息为：你已获得随机效果：生命值扣除【玩家自身.获得的未知卡片效果】

4.配置卡牌选择器控件参数

配置【页面设置】与【选项配置】的信息，并保存设置

5.制作一个宝箱作为触发元件，添加【选项卡】组件，并配置选项卡信息与触发选项区域

6.制作开启宝箱触发弹窗的节点图

**\*\*该节点图挂载在【宝箱】上**

|     |     |     |     |
| --- | --- | --- | --- |
|  |  | 要做的功能 | 对应的配置项 |
| 功能-1<br>玩家交互宝箱后，打开宝箱，并打开4选2弹窗 | 触发场景 | 交互选项后，触发事件 | 【事件节点】：选项卡选中时 |
| 条件 | / | / |
| 结果-1 | 打开4选2弹窗 | 【执行节点】：使用【唤起卡牌选择器】制作打开弹窗的效果<br>\*\*决策结果/选项对应列表=卡牌ID<br>\*\*决策数量上限/下限影响在本次选择中可选择的选项数量 |
| 结果-2 | 选项卡不能重复交互（避免宝箱被打开多次） | 【执行节点】：使用【激活/关闭选项卡】，将该交互选项关闭 |
| 结果-3 | 宝箱有打开的效果 | 【执行节点】：使用【设置预设状态】更改宝箱的状态 |

7.制作选择4选2弹窗卡片后的节点图

**\*\*该节点图挂载在【玩家】上**

|     |     |     |     |
| --- | --- | --- | --- |
|  |  | 要做的功能 | 对应的配置项 |
| 功能<br>选择选项后，获得对应选项的效果 | 触发场景 | 选择选项后 | 【事件节点】：卡牌选择器完成事件 |
| 条件-1 | 完成的是对应的卡牌选择器 | 使用双分支判断卡牌选择器索引<br>\*\*存在多个卡牌选择器时需要进行判断 |
| 条件-2 | 是玩家选择完成影响 | 使用双分支判断完成原因<br>\*\*避免由于超时等其他原因导致触发事件 |
| 条件-3 | 判断选择的卡牌选项是哪个 | 使用多分支判断选择结果<br>\*\*因为存在多选结果，返回的是列表，通过取列表的值判断 |
| 结果-1 | 确认选择的卡牌是哪个 | 由于4选2返回的是两个选项的列表，因此需要用到【列表迭代循环】+【多分支】判断选择的2个选项是哪个 |
| 结果-2 | 根据选择的卡牌执行对应结果-1<br>已知卡牌为添加特效 | 教程中以添加特效为例，实际制作时可按照需求自由制作不同效果 |
| 结果-3 | 根据选择的卡牌执行对应结果-2<br>未知卡牌为扣除生命值 | 教程中以【损失生命值】为例，并通过界面控件的文本框展示 |

8.将节点图分别挂载在元件上，保存设置

9.将制作完成的元件摆放在场景中，进入试玩即可体验该功能
