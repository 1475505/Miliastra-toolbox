# 阵营——区分团队的标签，影响攻击逻辑判定

## 功能示例-1：设定和玩家同阵营的造物（怪物）

### 1.概念简述

**阵营：** 通过配置不同阵营和阵营关系，用于节点图中的标签判断（是否x阵营），攻击逻辑判定（是否可以自主攻击某阵营单位）

### 2.教学内容

**功能效果：** 制作1个属于玩家阵营的造物（怪物），不攻击玩家而能帮助玩家攻击其他造物（怪物）（该功能可不使用节点图，仅面板配置实现），用于各类玩法中制作友善阵营的造物（怪物）/NPC、召唤物类技能

**效果演示：**

_\*\*如果在试玩时遇到造物（怪物）无法正常攻击、巡逻异常等问题，可以尝试使用【关卡设置-地形导航-烘焙网格体】_

### 3.操作步骤

**1.【局内编辑器】进入阵营配置页**

点击局内编辑器左上角（或直接按ESC键）>>>呼出左侧面板-ESC菜单栏，点击>>>在关卡设置中点击【阵营】

**2.【局内编辑器】配置面板参数**

按需配置阵营与阵营关系

_\*\*教程中沿用了默认未进行更改，仅做概念说明_

_\*\*配置项说明：_

_默认阵营对象：未进行过阵营修改的单位初始创建时的默认阵营_

_阵营关系：分为【敌对】和【友善】2种，可配置其他阵营对自身的关系和自身阵营对其他阵营的关系（如：阵营1对阵营2友善，阵营2对阵营1敌对，则实际游玩时，阵营2会攻击阵营1，但阵营1不会攻击阵营2）_

**3.【局内编辑器】创建同阵营造物（怪物）**

创建一个造物（怪物）（教程以普通丘丘人为例），在面板属性-阵营中，改为【初始玩家阵营】，保存设置

_\*\*教程中无额外功能需求，玩家为同一阵营，此处选择初始玩家阵营即可；如果玩家分为不同阵营，则需要额外考虑造物（怪物）具体归属是哪个阵营_

**4.【局内编辑器】放置造物（怪物），试玩体验**

将我们创建的造物（怪物）和常规造物（怪物）放置在场景中，进入试玩即可体验该功能

_\*\*如果在试玩时遇到造物（怪物）无法正常攻击、巡逻异常等问题，可以尝试使用【关卡设置-地形导航-烘焙网格体】_

_\*\*更多相关信息可自行查阅【综合指南-阵营】_

## 功能示例-2：阵营切换台，切换玩家阵营

**功能说明：** 玩家交互后，根据选项切换自身阵营

**效果演示：**

演示中放置了一个造物（怪物），根据造物（怪物）是否攻击玩家判断阵营切换是否成功

_\*\*演示中的文本显示，需达到Lv.3后解锁【自定义文本外显】功能，教程中以效果展示为主_

**实现步骤：**

1.创建一个元件，作为交互物

2.在该元件上添加【组件-选项卡】，并配置选项内容和触发选项区域范围

3.配置该元件的节点图（设置参考见下图），为达到预期的功能效果教程中配置如下：

|     |     |     |     |
| --- | --- | --- | --- |
|  |  | 要做的功能 | 对应的配置项 |
| 功能<br>交互选项后切换阵营 | 触发场景 | 选择选项后触发 | 【事件节点】：选项卡选中时 |
| 条件 | 具体选了哪个选项 | 【流程控制节点】：判断选项序号 |
| 结果-1 | 切换为造物（怪物）阵营 | 【执行节点】：修改实体阵营为造物（怪物）阵营 |
| 结果-2 | 切换为玩家阵营 | 【执行节点】：修改实体阵营为玩家阵营 |

4.将节点图挂载在元件上，保存设置

5.将元件放置在场景中，并放置一个造物（怪物）作为检测判断，进入试玩即可体验该功能
