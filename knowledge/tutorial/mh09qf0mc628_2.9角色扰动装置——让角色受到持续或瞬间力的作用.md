# 角色扰动装置——让角色受到力的作用

## 功能示例-1：进入区域后，垂直弹射

### 1.概念简述

**角色扰动装置：** 当靠近一定区域后，角色将会受到设定的力的作用而改变运动状态

_\*\*角色扰动装置仅对【角色】生效_

### 2.教学内容

**功能效果：** 制作弹弹菇，进入特定区域后受力的作用垂直弹射

**效果演示：**

### 3.操作步骤

**1.【局内编辑器】添加组件**

创建一个元件作为可弹射的物体（教程中以弹弹菇为例），在面板中点击>>>进入组件页，点击>>>选择【角色扰动装置】即可添加

**2.【局内编辑器】添加角色扰动装置**

打开【组件-角色扰动装置】后，默认不含有角色扰动装置，点击【+】即可添加

**3.【局内编辑器】配置面板参数**

添加【角色扰动装置】后可对装置类型、扰动规则、触发区域等进行调整，为达到预期的功能效果教程中配置如下：

|     |     |
| --- | --- |
| 要做的功能 | 对应的配置项 |
| 进入区域后触发 | 【触发区域】调整为球形，并配置合适的中心点与半径<br>_\*\*实际制作时可根据需要灵活调整_ |
| 将人物弹射升空 | 【装置类型】调整为弹射器 |
| 弹射方向为垂直向上 | 【扰动方向】设置为指定方向，并设置Y轴的方向向量与速率<br>\*\*实际制作时可根据需要灵活调整 |

_\*\*配置项说明：_

_稳定阶段时长(s)：角色持续受到角色干扰装置的时间，在此期间维持速率_

_衰减阶段时长(s)：角色受到的角色干扰装置，在扰动时间结束后，力的作用逐渐减少至0的时间_

**4.【局内编辑器】添加组件（\*\*此处配置仅加强表现，与实际功能作用无关）**

为了使弹射效果更加生动明显，玩家每次进入碰撞触发区弹射时，弹弹菇也有挤压弹射的变化；添加【组件-碰撞触发器】并调整为合适的大小（与角色扰动装置触发范围一致即可）

**5.【千星沙箱】节点图配置**_（\*\*此处配置仅加强表现，与实际功能作用无关）_

在实体节点图中，添加并配置该元件的节点图（设置参考见下图），为达到预期的功能效果教程中配置如下：

|     |     |     |     |
| --- | --- | --- | --- |
|  |  | 要做的功能 | 对应的配置项 |
| 功能-1<br>玩家进入碰撞触发区弹射时，弹弹菇有挤压弹射的变化 | 触发场景 | 玩家进入碰撞触发区后，触发事件 | 【事件节点】：进入碰撞触发器时 |
| 条件 | / | / |
| 结果-1 | 弹弹菇挤压弹射 | 【执行节点】：设置预设状态为变化值 |
| 功能-2<br>玩家离开碰撞触发区后，弹弹菇恢复 | 触发场景 | 玩家离开碰撞触发区后，触发事件 | 【事件节点】：离开碰撞触发器时 |
| 条件 | / | / |
| 结果 | 弹弹菇恢复 | 【执行节点】：设置预设状态为初始值 |

**6.【局内编辑器】挂载节点图**

将配置的节点图挂载在该元件上，保存元件设置，将制作好的元件放置在场景中，进入试玩即可体验该功能

## 功能示例-2：特定阵营玩家进入区域后，进行牵引传送

**功能说明：** 制作仅对特定阵营生效的角色扰动装置（传送器）

**效果演示：**

_\*\*教程为了方便演示增加了UI显示角色当前阵营（此处暂不拓展讲解）_

_\*\*演示中的文本显示，需达到特定奇匠等级后解锁【自定义文本外显】功能，教程中以效果展示为主_

**实现步骤：**

1.创建一个元件，作为交互物，添加【组件-角色扰动装置】，对装置类型、扰动规则、触发区域等进行调整，为达到预期的功能效果教程中配置如下：

|     |     |
| --- | --- |
| 要做的功能 | 对应的配置项 |
| 进入区域后触发 | 【触发区域】调整为正方体，并配置合适的中心点与缩放尺寸<br>\*\*实际制作时可根据需要灵活调整 |
| 将人物牵引传送到某区域 | 【装置类型】调整为牵引器 |
| 牵引方向为X方向 | 【扰动方向】设置为指定方向，并设置X轴的方向向量与速率<br>\*\*实际制作时可根据需要灵活调整 |
| 牵引中途可打断 | 【跳跃可打断】选项打开 |

2.【客户端节点图-过滤节点图】配置按照阵营过滤效果，为达到预期的功能效果教程中配置如下：

|     |     |     |     |
| --- | --- | --- | --- |
|  |  | 要做的功能 | 对应的配置项 |
| 功能<br>玩家为阵营1时，可被传送，玩家为其他阵营时无效 | 触发目标 | 查询进入角色扰动装置触发区的实体的阵营 | 【目标实体】=进入角色扰动装置触发区的实体<br>_\*\*该实体为角色，切换为玩家再查询阵营_ |
| 条件 | 判断进入角色扰动装置触发区域的玩家属于阵营1 | 运算节点：【是否相等】判断目标实体阵营是否为阵营1<br>_\*\*注意检查变量类型是否为【阵营】哦~_ |
| 结果 | / | / |

3.制作玩家切换阵营的功能_（\*\*此处配置仅用于单人测试演示不同阵营效果，对原功能无影响）_

此处以玩家施放技能切换进行简要演示说明

添加【技能】，对技能类型、动画编辑等进行调整，由于无动作特效等需求，因此使用【瞬发技能】

4.制作玩家切换阵营的节点图功能_（\*\*此处配置仅用于单人测试演示不同阵营效果，对原功能无影响）_

【客户端-技能节点图】配置一个通知服务器节点图，将其添加至技能中

5.将【技能】绑定在对应职业上

6.制作玩家切换阵营的节点图功能_（\*\*此处配置仅用于单人测试演示不同阵营效果，对原功能无影响）_

【服务器-实体节点图】技能施放后切换阵营

为达到预期的功能效果教程中配置如下：

_\*\*该节点图挂载对象为：【角色】_

|     |     |     |     |
| --- | --- | --- | --- |
|  |  | 要做的功能 | 对应的配置项 |
| 功能<br>施放技能后切换阵营 | 触发场景 | 施放技能后 | 【事件节点】：技能节点图调用时（接收客户端发送【通知服务器节点图】时触发） |
| 条件-1 | 判断玩家当前阵营 | 【流程控制节点】：使用双分支对“玩家阵营”进行判断 |
| 结果-1 | 玩家阵营为1时，切换为2 | 【执行节点】：【修改实体阵营】将【玩家】阵营改为2 |
| 结果-2 | 玩家阵营不为1时，切换为1 | 【执行节点】：【修改实体阵营】将【玩家】阵营改为1 |
| 结果-3 | 修改自定义变量 | 【执行节点】：设置自定义变量，将UI显示的玩家阵营切换为最新的阵营<br>_\*\*此处仅显示，与功能无关_ |

7.将节点图挂载在各对应元件/角色上，保存设置

8.将制作完成的元件摆放在场景中，进入试玩即可体验该功能
