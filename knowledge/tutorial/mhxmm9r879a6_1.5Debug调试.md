# Debug调试流程

### 操作流程

> **1、打开【日志】界面**
>
> 在千星沙箱中点击【窗口】>>>选择【日志】
>
> **2、选择需要检测的节点图**
>
> 在【日志】界面中勾选需要检测的节点图>>>点击【确认选择】
>
> **3、试玩并查看日志运行情况**
>
> 进入关卡试玩>>>触发需要检测的节点图事件>>>检查日志运行情况
>
> **4、进入调试模式，查看节点图运行情况**
>
> 点击日志中已运行的节点图>>>即可进入调试模式，查看运行中传输的数据和运行生效/未生效的节点图逻辑
>
> _\*\*如在下方演示中，可以通过调试模式看到触发事件时，【输出的具体参数】（事件源实体、事件源GUID、选项序号、选择者）；还可以看到节点图中【生效的逻辑树】（执行流-橙黄色连线、运行流-蓝色连线），未生效的逻辑树（灰色连线）_

_\*\*更多相关信息可自行查阅【综合指南-节点图日志】_

# Debug调试思路

### 问题原因排查思路

在实现的功能效果不符合预期时，需要进行节点图调试，主要检查两个情况：

> 1、【阻塞/没运行】是否节点图阻塞没运行；
>
> 2、【运行了但输出错误结果】是否节点图运行但参数输出错误

**排查流程：** 根据调试流程逐一配置 → 确认问题情况 → 修改调整，再次试玩调试

重复上述流程，直至功能表现符合预期

## 节点图阻塞/没运行

### 问题情况

常见问题类型为以下3种，将在后续进行逐一举例说明：

> ①未挂载节点图 或 节点图挂载对象无法监听对应节点图（对象错误 或 未配置相关组件）
>
> 对应的问题表现：Debug调试流程中，在【日志】勾选对应的节点图后，试玩操作后日志列表为空
>
> ②使用了错误的事件节点 或 试玩时触发事件的操作和配置的事件节点不统一
>
> 对应的问题表现：Debug调试流程中，在【日志】勾选对应的节点图后，试玩操作后日志列表为空
>
> ③输入了错误的参数，导致条件判断错误，后续的执行节点不运行
>
> 对应的问题表现：运行节点图的调试模式中存在未生效逻辑树

### 问题示例1：未挂载节点图

**问题案例：** 制作了一个选项交互功能的节点图，【选项卡交互后】触发事件xxx，但没有在对应交互的物件上挂载节点图

**问题原因：** 制作了实体节点图，但没有挂载在对应实体上

**检查&修改方式：** 检查触发事件的实体是否配置节点图 → 若未配置，则在实体上配置对应的节点图

### 问题示例2：节点图挂载对象错误

**问题案例：** 制作了一个UI按钮交互后触发执行特定功能的节点图，【界面控件组触发时】执行xxx，但挂载节点图挂给了非玩家的其他单位（该事件仅能被“玩家”监听）

**问题原因：** 制作了实体节点图，但挂载对象无法监听到该节点图的事件

**检查&修改方式：** 确认所使用的节点图的【事件节点】是什么 → 确认被挂载对象能否监听该事件 → 如果不能，则更换正确的挂载对象

_\*\*该问题类型涉及不同事件节点的监听规则，此处以类型说明为主，不做拓展讲解_

### 问题示例3：节点图挂载对象未配置相关组件

**问题案例：** 制作了一个碰撞某区域触发执行特定功能的节点图，【进入碰撞触发器时】执行xxx，但没有在对应交互的物件上挂载【碰撞触发器】组件

**问题原因：** 制作了实体节点图，但挂载对象没配置相关组件，无法正常触发事件

**检查&修改方式：** 确认所使用的节点图的【事件节点】是什么 → 确认该事件是否涉及特殊的组件（如：选项卡/碰撞触发区/命中检测等） → 如果涉及但未配置，则在实体上配置对应的组件

_\*\*该问题属于面板配置问题，并不涉及节点图改动，此处仅作示意说明_

### 问题示例4：使用了错误的事件节点

**问题案例：** 预期实现的功能为【选择选项后，执行xxx】，节点图配置的事件为非【选项卡选中时】的事件节点

**问题原因：** 使用的事件节点与实际功能所需的不匹配

**检查&修改方式：** 确认所使用的节点图的【事件节点】是什么 → 确认该事件是否与预期功能的触发事件一致 → 若不一致，则更换对应的事件节点（如：选中选项触发xxx，使用【选项卡选中时】；进入碰撞区触发xxx，使用【进入碰撞触发器时】）

### 问题示例5：试玩时触发事件的操作和配置的事件节点不统一

**问题案例：** 预期实现的功能为【选择选项2后，执行xxx】，试玩时没有选择选项2

**问题原因：** 试玩时操作未正常触发配置的事件节点

_\*\*该问题属于个人试玩操作失误（非面板配置/节点图错误），此处不做额外演示，仅给出调试排查思路方向_

### 问题示例6：输入了错误的参数，导致条件判断错误，后续的执行节点不运行

**问题案例：** 预期实现的功能为【交互选项后，检查玩家身上的变量，如果满足条件则执行xxx】，实际输入变量不符合预期，导致条件判断错误，无法正常执行（预期走【是】的分支）

**问题原因：** 节点图编写存在错误，导致运行中输入/输出结果不符合预期

**检查&修改方式：** 在日志中选择触发的事件进入调试模式 → 逐一检查运算流中具体输入/输出参数是否符合预期 → 如果出错，修改节点图中相关配置

_\*\*该问题类型与下方课程的输出错误结果相关，将在后续展开讲解，此处仅作简单示例，提供检查思路_

## 节点图运行了但输出错误结果

### 问题情况

常见问题类型为以下2种：

> ①变量类型错误，导致输出错误的结果
>
> ②运算存在问题，导致输出错误的结果
>
> 以上2种类型对应的问题表现均为：在调试模式中输出的参数错误（类型错误、数值错误）

### 问题示例1：变量类型错误，导致输出错误的结果

**问题案例：** 设置的变量类型为【整数】，但节点图中使用了【浮点数】，实际运算流中输出结果错误

**问题原因：** 使用了不匹配的变量类型，输出结果不符合预期，无法正常执行

**检查&修改方式：** 在日志中选择触发的事件进入调试模式 → 检查运算流中未执行的前置输入/输出参数是否符合预期 → 如果出错，修改节点图中相关配置

_\*\*节点图是一种【强类型】语言，会严格按照变量类型进行运算和判断，即节点图中的类型与实际面板配置类型不一致就会出错_

### 问题示例2：运算存在问题，导致输出错误的结果

**问题案例：** 预期实现的功能为【交互选项后，玩家变量值+1，检查玩家身上的变量=5时，执行xxx】，但连接节点图时选择了错误的入参值，在玩家积分为4时就已经执行

**问题原因：** 节点图编写存在错误，导致运行中输入/输出结果不符合预期

**检查&修改方式：** 在日志中选择触发的事件进入调试模式 → 逐一检查运算流中具体输入/输出参数是否符合预期 → 如果出错，修改节点图中相关配置
