为了构建对抗性玩法而将实体划分为不同的群体，归属于同一群体内的玩家有着相同的游戏目标，这个群体被称为_阵营_

在组队竞技类型的游戏中，阵营可以作为结算目标或得分的判断依据

# 一、阵营的全局定义

在_基础信息_上选择阵营之前，需要先在关卡设置的阵营页签定义玩法所需要的阵营

_阵营名称_：该阵营的命名

_默认阵营对象_：新创建的玩家/物件/造物将被归类到对应的默认阵营中

_包含的玩家_：是一个玩家的列表，代表初始有哪些玩家归属于该阵营

_包含的实体_：是物件和造物的列表，代表初始有哪些物件和造物归属于该阵营

# 二、阵营关系

任意两个阵营之间都一定存在_敌对_和_友善_两种关系，可以通过关卡设置中的_阵营关系_进行配置

需要注意的是，阵营关系是单向的，阵营1对于阵营2敌对，但阵营2不一定对阵营1敌对。可能存在阵营2对阵营1友善的情况，此时仅阵营1的实体可以攻击阵营2，阵营2的实体无法攻击阵营1

# 三、默认阵营配置

每个阵营均可以配置一个_默认阵营对象_。当该对象类型的实体（例如图中的玩家和物件）创建时，如果是默认阵营（阵营的值为0），那么会被分配至该实体类型的默认阵营

# 五、经典模式中的阵营

相比于超限模式，经典模式中的阵营会受到一定的限制：

1.所有玩家和角色为【玩家阵营】，不可修改

2.所有造物为【造物阵营】，不可修改

3.物件可以配置为【玩家阵营】【造物阵营】【物件阵营1】【物件阵营2】【物件阵营3】

4.所有实体不可动态修改阵营，因此无法使用阵营修改等相关执行和事件节点

5.阵营间的关系不可修改

**经典模式中的各个阵营关系如下**

横列为发起方，纵列为接受方

|     |     |     |     |     |     |
| --- | --- | --- | --- | --- | --- |
|  | **玩家阵营** | **造物阵营** | **物件阵营1** | **物件阵营2** | **物件阵营3** |
| **玩家阵营** | 友善 | 敌对 | 敌对 | 敌对 | 友善 |
| **造物阵营** | 敌对 | 友善 | 友善 | 友善 | 敌对 |
| **物件阵营1** | 敌对 | 敌对 | 友善 | 敌对 | 友善 |
| **物件阵营2** | 敌对 | 敌对 | 敌对 | 友善 | 友善 |
| **物件阵营3** | 友善 | 敌对 | 友善 | 敌对 | 友善 |

各阵营之间的敌对关系可用下述有向图表示

物件阵营3通常用于与玩家阵营友善且可以辅助角色攻击造物的物件上。而物件阵营1用于常规物件，如果需要与物件阵营1敌对的物件，一般将物件阵营2用于这类物件。
